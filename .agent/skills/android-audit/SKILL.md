# android-audit

Use when changing Android application behavior.

Checklist:
- Activity/Fragment lifecycle correctness.
- Runtime permission handling.
- Android SDK 29+ behavior.
- Android 13-16 behavior.
- Scoped Storage compatibility.
- MediaStore and SAF correctness.
- Background execution limits.
- Foreground Service requirements.
- OEM restrictions and battery policy.
- Process death and configuration changes.
- No unnecessary dependencies.

Output:
- Findings.
- Risk level.
- Minimal fix.
- Validation steps.
