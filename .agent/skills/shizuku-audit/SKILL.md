# shizuku-audit

Use when touching Shizuku integration.

Checklist:
- Binder lifecycle.
- Permission state.
- Service reconnection.
- Shizuku not installed.
- Shizuku stopped.
- User denies permission.
- Startup delay.
- Multi-user behavior.
- Shell UID 2000 behavior.
- SELinux context.
- Fallback path.

Output:
- Failure modes.
- Minimal fix.
- Logs to verify.
- Device test matrix.
